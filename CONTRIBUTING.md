# Contributing to McWritely

Thank you for your interest in contributing to McWritely! We welcome all contributions, from bug reports to feature enhancements.

### 1. Identify an Issue or Feature
- Look through our [issues](https://github.com/anton-g-kulikov/mcwritely/issues) or open a new one to discuss your ideas.
- For larger changes, it's always best to open an issue first to discuss the design.

### 2. Fork & Branch
- Fork the repository and create a branch for your work.
- Use a naming convention like `feat/short-description` or `fix/short-description`.

### 3. Build & Test
- Run `./build.sh` to ensure the project builds locally.
- Run `./test.sh` (if available) to verify your changes.

### 4. Code Style
- Follow existing Swift style and naming conventions.
- Keep changes focused and small where possible.
- Use descriptive variable and function names.

### 5. Commit Messages
- Use [Conventional Commits](https://www.conventionalcommits.org/) (e.g., `feat:`, `fix:`, `docs:`, `chore:`).

### 6. Submit a Pull Request
- Open a PR against the `main` branch.
- Provide a clear description of your changes and why they are needed.
- Link any related issues.
